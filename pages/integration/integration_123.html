INTEGRATION 123 - Data Pipelines
Integration and Data professionals have been creating EAI (Realtime) and ETL (Batch) jobs for a long time. Cloud computing in the past decade has evolved the products and tools for professionals on both sides. Let’s gear up a bit for the next wave with data pipelines.

Data Ingestion

  • ETL (Extract, Transform, Load): Perform data transformations on the source data and load the transformed data into target systems like Data Warehouses.
  • ELT (Extract, Load, Transform): Load the source files as raw data in cloud Data Lakehouse and uses the transformation engine of Analytics Services.
  • Data Pipelines benefits data quality, automation, and intelligent insights, which will usually lead to better customer centricity.
  • Data Pipelines address usecase and scenarios in Visualization and Machine Learning.
  • Data Handling should use 3Vs (Velocity, Volume, and Variety) to define the non function requirements from ingested data sources.
  • Types of data: structured, semi-structured, unstructured. Relational Databases are structured, Document Databases are semi-structured, and Cloud-based Object Storage is unstructured.
  • Data Pull Style: Batch and Streaming
    • Batch Mode: Daily or Monthly Accounting Processes. Requires High Throughput Solutions with configurable batch options.
    • Streaming Mode: Click Streams, Machine Data. Requires Low Latency, Fault-Tolerant Systems.
  • Data Processing Systems are categorized based on Transactional and Analytical Workloads
    • OLTP: Online Transaction Processing System. Eg: Order Entry, Payment Processing.
    • OLAP: Online Analytical Processing System. Eg: Forecasting, Scoring, Predictions.
  • OLTP systems are at the beginning of the pipeline and mostly high volume short transactions. OLTP system prioritizes writes and maintains integrity with relationship and ACID transactions.
  • OLAP systems are designed specifically for querying and analyzing large datasets. Eg: Data Warehouses, Data Lakes.
  • Most Enterprises use OLTP for their transactional applications while also periodically loading data into OLAP for analysis.
  • Columnar Style Storage pattern allows better compression, fast reads on column values and are more preferred for OLAP systems.
  • There are additional formats like Avro and Parquet for unstructured text and metadata. Requirements on schema evolution, compression, read/write performance determine the choice of one over the other.
  • Batch Data Pipelines that require high throughput use parallel processing to distribute data across multiple nodes in a cloud cluster.
  • Data models evolve from concept (domain), logical (entities and relationships), and physical (datatypes in database stack) layers.
  • Data cleaning techniques are applied to the data extracts from several sources and mapped to the model layer.
  • Let's start with a free tier access on AWS, and create a simple mule traget side service api that will push an object on an AWS S3 bucket.
  • Batch Processing scenarios with large volume and high throughput provision on-demand compute resources.
  • Stream Processing scenarios with high velocity that require low latency or near real-time analytics provision and run compute resources continuously.
  • Batch Ingestion Scenarios on AWS platform can use services like AWS Data Migration and AWS Data Sync.
  • Streaming Ingestion Scenarios on AWS platform can use services like AWS Kinesis and AWS Lambda. An example use case can be capture audio and video files to enhance physcical security posture or send application logs to observability stacks like Elastic or Splunk.
