INTEGRATION 123 - Data Pipelines
Integration and Data professionals have been creating EAI (Realtime) and ETL (Batch) jobs for a long time. Cloud computing in the past decade has evolved the products and tools for professionals on both sides. Let’s gear up a bit for the next wave with data pipelines.

Data Ingestion

  • ETL (Extract, Transform, Load): Perform data transformations on the source data and load the transformed data into target systems like Data Warehouses.
  • ELT (Extract, Load, Transform): Load the source files as raw data in cloud Data Lakehouse and uses the transformation engine of Analytics Services.
  • Data Pipelines benefits data quality, automation, and intelligent insights, which will usually lead to better customer centricity.
  • Data Pipelines address usecase and scenarios in Visualization and Machine Learning.
  • Data Handling should use 3Vs (Velocity, Volume, and Variety) to define the non function requirements from ingested data sources.
  • Types of data: structured, semi-structured, unstructured. Relational Databases are structured, Document Databases are semi-structured, and Cloud-based Object Storage is unstructured.
  • Data Pull Style: Batch and Streaming
    • Batch Mode: Daily or Monthly Accounting Processes. Requires High Throughput Solutions with configurable batch options.
    • Streaming Mode: Click Streams, Machine Data. Requires Low Latency, Fault-Tolerant Systems.
  • Data Processing Systems are categorized based on Transactional and Analytical Workloads
    • OLTP: Online Transaction Processing System. Eg: Order Entry, Payment Processing.
    • OLAP: Online Analytical Processing System. Eg: Forecasting, Scoring, Predictions.
  • OLTP systems are at the beginning of the pipeline and mostly high volume short transactions. OLTP system prioritizes writes and maintains integrity with relationship and ACID transactions.
  • OLAP systems are designed specifically for querying and analyzing large datasets. Eg: Data Warehouses, Data Lakes.
  • Most Enterprises use OLTP for their transactional applications while also periodically loading data into OLAP for analysis.
  • Columnar Style Storage pattern allows better compression, fast reads on column values and are more preferred for OLAP systems.
  • There are additional formats like Avro and Parquet for unstructured text and metadata. Requirements on schema evolution, compression, read/write performance determine the choice of one over the other.
  • Batch Data Pipelines that require high throughput use parallel processing to distribute data across multiple nodes in a cloud cluster.
  • Data models evolve from concept (domain), logical (entities and relationships), and physical (datatypes in database stack) layers.
  • Data cleaning techniques are applied to the data extracts from several sources and mapped to the model layer.
  • Let's start with a free tier access on AWS, and create a simple mule traget side service api that will push an object on an AWS S3 bucket.
  • Batch Processing scenarios with large volume and high throughput provision on-demand compute resources.
  • Stream Processing scenarios with high velocity that require low latency or near real-time analytics provision and run compute resources continuously.
  • Batch Ingestion Scenarios on AWS platform can use services like AWS Data Migration and AWS Data Sync.
  • Streaming Ingestion Scenarios on AWS platform can use services like AWS Kinesis and AWS Lambda. An example use case can be capture audio and video files to enhance physcical security posture or send application logs to observability stacks like Elastic or Splunk.

Data Transformation

  • Often raw data extracts have to cleansed before it can be used for model training or can generate actionable insights. Raw extracts can have simple issues like duplicate records, records with fields having null values, inconsistent date and decimal formats. Some features may require data encryption, and use of aggregates and filters.
  • Some of these transformation requirements can be implemented while ingesting data into the data warehouses and data lake.
  • Big Data processing requires experience Apache Hadoop, Apache Spark. Amazon EMR is a managed cluster platform that simplifies the provisioning and operation tasks.
  • For the datasets that are not categorized as big data, we can use the transformation capabilities in other ETL, EAI, scripting engine to deliver cleansing tasks in data pipelines.
  • An example can be that we can pull a large dataset of product and pipeline filter operation can help label the top products in visualizations.

Data Storage

  • Over the years, relational databases are the backbone of structured datasets created as part of OLTP and OLAP systems.
  • Cloud Computing has created other set of options for storing all kinds of data encompassing not just structured but semi-structured and unstructured datasets.
  • Architects consider these cloud-based storage options to build solutions that are optimized for cost and performance.
  • Storage cost and access timeframe varies based on the temperature of the data, which can be hot, warm, and cold.
  • A large number of software companies have SaaS products and PaaS tools that provide direct secure access to the data stored in the public cloud.
  • One can expect that data pipelines will read/or write data from/or to databases (on-premise / or cloud) and othe cloud storage services (s3, azure blob, gcp object storage).

Data Analytics

  • Typically, the job of a data pipeline is to get the data ready for business analysis.
  • At times, data pipelines implements 3 -layered solution to process data through different layers before it can be consumed by business analytics and reporting tools.
    • Bronze Layer: Raw Data Ingestion
    • Silver Layer: Clean and Conformed Data
    • Gold Layer: Business Level Aggregates
  • Analytics tools offer several features to slice and dice facts across several dimensions to generate business intelligence. Analysts work with platform services like Amazon Redshift, Apache Spark on EMR, Azure SQL Server, Power BI to create reports and dashboards
  • Apache Flink can use Java, Scala to analyze the stream of data in real-time. Amazon Athena is an in-memory query engine that helps you analyze data ingested in S3 datalakes. One can also load and unload data from S3 to Redshift clusters.

Understanding Well-Architected Side in Cloud Solutions

  • Data pipelines, when created as part of large digital transformation, require professionals to rely on a well-architected framework to realize business value in cloud programs.
  • AWS has created a well-architected framework that focuses on key concepts, design principles, and architectural best practices for designing and running workloads in the cloud.
  • Well-Architected Framework addresses concerns related to security, reliability, performance, cost, and sustainability.
  • List down different framework pillars and continuously learn the best practices in each pillar.
    • Operational Excellence Pillar: This pillar is focused on logging and monitoring and continuous improvement on root cause analysis with end to end pipeline traceability from source to destination.
    • Security Pillar: This pillar extends the principle of least privilege with requirements like secure access to data apis with policy enforcement and logging unauthenticated attempts.
    • Reliability Pillar: This pillar discusses applying queuing architecture to prevent failed connections. It futher extends monitoring connection pools with usage alerts.
    • Performance Efficiency Pillar: This pillar keeps the automation at the center and requires us to use tools that improve agility and productivity. For eg: Automated CI/CD for build and deploy pipelines, monitoring service latency and throughput.
    • Cost Optimization Pillar: This pillar ensures that cost management is integral to solution design. An example may be that Invoice matches to purchased resources and reconciles to the approved expense. Cloud products have built-in billing dashboards, and the detailed usage data can be pulled into custom reports.
    • Sustainability Pillar: This pillar extends baseline cost management at the solution level and collects resource-level usage to determine the sustainable path for the scaled deployment of the solution.
