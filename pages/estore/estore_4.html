LEAD FEATURE

As a L2C Stakeholder, I would like to see that all the Leads that are qualified in our Marketing Application should be created in Salesforce Sales Cloud within an hour. I also require that when Sales Rep convert the lead to create account, contacts and opportunities, we should have an extensible Lead Conversion Engine that relies on standard Convert Lead feature of Sales Cloud. Mulesoft API Implementation Flows should allow us to build Conversion Engine with enterpise specific requirements, rules and processes that we can evolve as we scale and integrate within the realm of enterprise applications for our clients. All these automation should be triggered through a Mulesoft API resource so that external calls to the Salesforce Lead Object from all internal and external applications should be unified and standardized through an API Layer.

In the initial sprint, the acceptance criteria is to have a system level API application with Create Lead, Convert Lead and Delete Lead resources. Delete Lead is only for clean-up and should be protected with a special username / password if we are using basic auth or with a special scope if we are using token based auth schemes.

In subsequent sprints, we will have to build process level API resources:

  • Automation to create leads should run every hour that will call on Create Lead system level resource.
  • Implement all the business rules in the Lead Conversion Engine that will call on Convert Lead system level resource.
  • Both automation should use common batch framework that we have to build and test as we integrate system level resources in process layer. We have different approaches and mule components that support Batch processing like Pub-Sub patterns, Mule Batch componenet with Asynchronous Parallel Processing, Parallel For Each so we have to apply fit for purpose analysis which can be backed with solution views, POCs and then use the one that gets approved in specific implementations.

PRODUCT AND PRICEBOOK FEATURE

As a L2C Stakeholder, I would like to mention that we sell both internationally and in US, so our Salesforce Admin has activated multiple currencies in Salesforce Org Admin has added GBP - British Pound, EUR - Euro, JPY - Japanese Yen,SGD - Singapore Dollar along with USD - US Dollar but the corporate currency is USD - US Dollar. Our product group creates and mantain product sheets to discuss product codes, pricing and other market level product configurations. I would like Automation team to pick up the product sheets on hourly schedule and create / and update products in the Product2 salesforce object and also create the associated pricebook entries in PricebookEntry salesforce object. The pricebook in Pricebook2 salesforce object are created and managed by the product architect manually through salesforce UI. Please take a note that Product Code is how we are going to uniquely idenfiy each product.

In the initial sprint, the acceptance criteria is to have a system level API implementation for Create Product, Retrieve Product and Delete Product resources. Also we require Create., Retrieve and Delete for Pricebook Entry. Delete resources are required only for clean and should be protected with special access.

In subsequent sprints, we will have to build process level API resources:

  • Automation to create products should run every hour and process level composition should call on both product and pricebook system level resources to setup the product code along with unit price in the salesforce org.
  • Any Errors related to setup issues like missing Pricebook should sent an alert to product group immediately. If the same error is picked in the next hourly run, the notfication alert counter should be incremented as well.

OPPORTUNTIY AND OPPORTUNITY PRODUCT FEATURE

As a L2C Stakeholder, I would like to see that when we convert both the leads (John Doe and Jane Doe), we should have accounts, opportunity and contact gets created through our extensible Lead Conversion Engine. After the conversion on both leads, we should have both the opportunities: BSofSW - Ground-mounted Panels; Saito - Ground-mounted Panels in our Salesforce Org.

As a Team Lead, I need the capability to update the pricebook, currency and amount on the opportunity. I should then able to add the products to the opportunity with an API resource by passing in the opportunity name along with product list. Product in the list should require product code and quantity whereas the API implementation should pick up the unit price from the price book entry.

In the initial sprint, the acceptance criteria is to leverage on Convert Lead system level API resource to create an opportunity; create implementation for Retrieve Opportunity, Update Opportunity, Create Opportunity Product and Remove Opportunity Product resources.

In subsequent sprints, we will have to build process level API resources to refine the business processes with Opportunity object and generate business outcomes as per stakeholder reviews.

QUOTES AND QUOTE LINES FEATURE

Sales Reps will have several opportunities in pipeline, and opportunities will move through several stages in a sales path. Also reps will create several quotes in the negotiation phase and one of the quote will be selected in a closed/won scenario which will be a primary quote. As L2C stakeholder works with the product groups, there are scenarios that require customization and automation at business process layer and requires team to add a capability to create quotes in a nightly batch with the list of opportunites in an excel.

As a Team Lead, I need an API resource that will create an quote by passing opportunity name and quote name.

In the initial sprint, the acceptance criteria is to add the quote resource as a nested resource in the opportunity top level resource. When the quote is create with automation, the quote lines will be created with opportunity products. We will rely on salesforce sync feature when the opportunity / or quote changes.

In subsequent sprints, as the requirements elaborate, we will design the customization on sync feature.

ORDER AND ORDER PRODUCT FEATURE

L2C stakeholder mentioned that sales repo close about 200 opportunities on a given day and provisioning group would like to see orders for all the opportunites that are in closed won status. One of the lead account executive will export an excel with the opportunities in closed won status along with the accountname, contract start date and contract term.

As a Team Lead, I need an system layer API resource that will create a contract by passing account name and contract details.

As a Team Lead, I need another system layer API resource that will create an order by passing in the contract number and opportunity name.

We also need to provide the capability to add and remove products from the order so we need anpther 2 system level resources. As this step is done manually by sales rep for the time being, we need a process layer test capability for these 2 resources of adding and removing order products.

In subsequent sprints, as the requirements elaborate, we will design the process layer api that will read the daily opportunity extract excel and automates the contract and order creation in a single step by calling on system level resources.
