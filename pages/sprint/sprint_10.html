Admin Intermediate Trailhead

Admin Intermediate: Completed 11/13/25

  • In the Admin beginner trail, we learn to navigate with salesforce apps and platform, install learner packages like dreamhouse, customize the lightning experience and build reports and dashboards.
  • In the Admin Intermediate trail, we start to get into point and click solutions. We can add formulas, roll up summary fields to objects for things how long the opportunity is open, sum of opportunities for an account. Objects have a validation rule features which helps you with field level rules, conditions like zipcode for contact should be same as related account zip code.
  • With Data security module, we start to learn that platform offer different levels through which admin can control access. Different levels are org level, object level, field level and even the record level. Profiles are the main tool through which access is granted but there are things like Permission Sets, Role hierarchies and Sharing rules through which access to data and objects can be further managed.
  • We learn how to configure picklists so that users can pick the correct value from the dropdown and data entry issues.
  • We also build a custom apps like suggestion box and space station apps. In these modules, we build with techniques learned in data modeling, lightning experience customization to create a lightning app with point and click solutions.
  • With features like email templates, global actions and approval processes, we can automate tasks like converting prospects to customers when prospects meet the entry criteria.
  • In App Exhange basics, we learn how to navigate and search for solutions available through partner ecosystem and install them for customers.
  • With external services, we can import API specs from partners to create invocable actions in an apex service which can then natively be called from tools like flow builder.
  • Though salesforce mobile app is offered along with the platform, there are special considerations when we rollout it to the users. The way users use mobile devices are quite different as we have to capture micro moments, start with sponsors and stakeholders, go initially with low hanging fruits and quick wins, work with change management, marketing and drive adoption.

Developer Intermediate:

  • Card-1: Complete the following trail step, Flow Builder Basics: One of ways we can automate tasks in salesforce and create point and click business automation is with Flow Builder. Flow Builder let you create both Interactive experiences with feature like Screen Ffows, Autolaunched flows and trigger based automation with features like Record Triggered, Schedule Triggered etc. Flow Builder has building blocks in form of Interaction Elements (Screen), Data Elements (Create Records, Get Records) and Logic Elements (Assignment, Decision). We create resources for things like variable and formulas.
  • Card-2: Complete the following trail step, Data and Actions in Flows: As users work on record details and save them, we create trigger based flows which can be configured with record selection criteria and uses data elements like get records, create records. Data Elements like Get Records create variable which holds the data record values which can be aggregated to create records in related objects or actions like email, alerts and chatter post. We can reference the running user profile and fields values with global variables.
  • Card-3: Complete the following trail step, Flow Builder Logic: We build the logic in the flows with the decision elements to create multiple paths , use assignment operators to copy values in the variables, reuse common tasks with subflows and use permissions and global variables to create different outcomes.
  • Card-4: Complete the following trail step, Record Triggered Flows: Create a Review Opportunity with Account Owner flow which is based on the opportunity object, and we have specified the conditions. It creates an email task for the account owner. It also has a scheduled path which gets trigger 3 days after the last modified date on the opportunity.
  • Card-5: Complete the following trail step, Autolaunched and Scheduled Flows: Create a Clone Closed Case flow which is an autolaunched flow which clones the case and we trigger it with the action button on the case record.
  • Card-6: Complete the following trail step, Screen Flows: Complete a Create Case for Contact has the couple of screen elements that covers elements like textarea, toggle, radio and checkbox components. It then uses create record element to create a case record based on input data from screen components.
  • Card-7: Complete the following trail: Prepare for Salesforce Releases

Customers generally explore release features with preview instances and promote customizations with non-preview instances in the release months. It is suggested to use filters to go through release notes and quick wins features are something easy to start for release managers.

    Releases generally require a bit of planning with focused approach and it is recommended in trail: Forward Plan; Roles, Responsibilities and Release Processes, Advance Tools

    Understand the gaps in the released features and how it will be absorbed in the existing UIs and business processes. Depending upon complexity of the org salesforce environment and number of licensed users, you may need to define roles, responsibilities, apply testing approaches covering unit, integration, uat and create advance tooling in the form of spreadsheets with release matrixes or custom object to list new features. Release Teams also have to make sure adoption of new features.

    Some of the Einstein and Platform features for Spring'25 are AI Assisted Agent Generation, Testing and Policy Center. Mulesoft Agentforce features include Topic Center, AI Chain Connector. Data Cloud can use Unstructed Audi and Video Content.

Get Ready For Agentforce

Get Ready For Agentforce: Completed 11/14/24
  • Card-1: Complete the following trail step: Generative AI Basics. As we go through the readout about the capabilities and evolving technology ecosystem of generative AI, we get to know terms like Natural Language Processing (NLP), Large Language Model (LLM) which uses machine learning alogirthms, neural networks, training with enormous datasets and accelerated computing chips to provide us with AI applications like ChatGPT. We understand that AI application are going to change the way we work and there is an emerging ecosystem computer hardware (Nvidia), hyperscalers / public cloud, foundation models (GPT, LLaMA), infrastructure optimization and training and applications like ChatGPT, Github Copilot, Einstein Copilot.
  • Card-2: Complete the following trail step: Discover Agentforce Agents. In this module, we understand that Agentforce allows salesforce customers to build AI agents that can be deployed across all salesforce products to drive customer success. Agents uses LLMs and business data in customer 360 and external data pulled in Data Cloud along with established guardrails to work as Service Agents, Sales Development Representative (SDR), Sales Coach, Merchandiser, Buyer Agent, Personal Shopper, Campaign Optimizer to, autonomously perform tasks and help customers alongside your reps and teams. Agentforce is build on salesforce reasoning engine and action framework that uses industry LLMs, platform components, knowledge articles, public data and guardrails configurations to augment customer salesforce experience.
  • Card-3: Complete the following trail step: Agentforce Service Agent. We can see that Agentforce service agent can be deployed with ease and can help service reps with 24/7 support, generating clear instructions and responses and providing smoother handoff in difficult conversations all while being grounded with trusted data.
  • Card-4: Complete the following trail step: Configure an Agentforce Service Agent. You have to start by create an Agentforce playground and follow the instructions to enable the einstein features, publish a digital experience site of Coral Clouds, use agent builder to configure coral cloud service agent with topic, actions and instructions. You will then add the coral cloud agent to the coral cloud experience site using the Embedded Messaging component. You will test the deployed agent by accessing the site with published url using the email address and membership number provided in the module readout.
  • Card-5: Complete the following trail step: AI Strategy.
    • As we start to read, we understand that organization have to formulate an AI vision which will require an AI center of excellence, and participation from all business lines as AI is a tectonic shift which though in short term require pilot projects to realize quick wins with simpler use cases and improve on key business metrics but the long term goal is to create AI powered products and services that generate new business frontier. It is imperative that organization require a data strategy and data products to realize success on AI initiatives.
    • After the organization commits to AI frontier, the AI governance readout explains that there are risks like inaccuracies and bias, security breaches and the pace with AI revolution. Salesforce mentions responsible, accountable, transparent, empowering and inclusive as their AI trusting principles. AI governance org has to look into regulation landscape, monitor AI usage and manage the AI inventory along with mitigating several types of risks like data & privacy.

  • Card-6: Complete the following trail step: AI + Data: Project Planning.

  • Card-2: Identity Resolution Ruleset We will the create a ruleset that will match source records from Individual object into unified individual profiles and store them in Unified Individual Object and its related unified contact point object. The ruleset will include following match rules:
    • Fuzzy Name and Normalized Email
    • Fuzzy Name and Normalized Phone
    • Fuzzy Name and Normalized Address
  • Card-3: Calculated Insight We can then add another data stream for case which will add cases to the Data Object Model, and then relate the case DMO to other DMO objects. We can then build a calculated insight with unified individual ID, case status and aggregated case count as dimensions.

Unlock Your Data with Data Cloud

  • Card-1: As we being to do trail steps, we understand that Data Cloud harness the cloud infrastructure, streaming and batch frameworks to unify the enterprise data pulled in from salesforce orgs, data lakes, and other structured and non-structured data sources into a customer 360 data model.
    Organizations can then leverage the capabilities of salesforce lightning platform, features of Einstein AI engine and acquired capabilities of Mulesoft / Tableau to deliver enriching experiences in Sales, Service, Marketing and Commerce cloud.
    As we being to do trail steps, we understand that Data Cloud harness the cloud infrastructure, streaming and batch frameworks to unify the enterprise data pulled in from salesforce orgs, data lakes, and other structured and non-structured data sources into a customer 360 data model.
  • Card-2: As you move along in the trail, you can then dive into the technical side of Data Cloud architecture and understand that Data Cloud is combining CRM + AI + Data + Trust in salesforce next gen platform capabilities.
    • Connect all your data sources whether batch or streaming through Data Streams.
    • Transform, Govern and hormonize to customer 360 data model with Data Model Objects (DMO).
    • Create Identity Resolution Rulesets to unify the customer 360 model to get to a normalized view of customer profile.
    • Data explorer and profile explore can show the segregated view of harmonized and unified data in a dataspace.
    • With analytics, insights, AI predictions, segmentation, the unified data can then be targeted to various actions, flows, activations and exports.
    • With the capabilities in Einstein 1 AI platforms, we can enable AI experiences in Sales, Service, Marketing and Commerce cloud.
  • Card-3: Admins will setup up the Data Cloud, create users and assign permission sets and depending upon the org factors, you will have a data cloud org provisioned as a new org or an in an existing salesforce org. Irrespectively you will have to proceed to next step of connecting other salesforce org and data sources through external integrations setup tasks, installing data kits, installing packages that creates data streams, DLOs, DMOs. We can then create additional Data Cloud artifacts which can include additional data streams, DLOs, DMOs, Calculated Insigts, Segments, Activation Target and Activations.
    To jump start the learning, we can connect to “Trial OrgFarm For U” salesforce org which has preconfigured standard data bundles for Sales Cloud, Salesforce CDP CRM Loyalty, Salesforce CDP Salesforce Commerce, Service Cloud.


    You can install one of these data bundles by creating a new data stream and selecting Salesforce CRM as a connected sources. When you install a Sales Cloud data bundle, you will see that Sales Cloud package with publisher as CDP CRM 1 is installed in the Installed Packages Setup and SalesDK data kit is installed in the Data Kits Setup.


    You can then see that Data Streams like Account_Home, Lead_Home, Opportunity_Home that are part of Sales Bundle are created and viewable in the list view of Data Streams tab of Data Cloud App.

    Along with Data Cloud setup and importing standard bundles, you need to prepare the implementation with Data Discovery by identifying other data sources like AWS S3, Data Lakes which will become additional data streams. You also need to work with Data Ethics team to follow data governance and consent procedures that will cover things like GPDR and other data residency requirements. After that comes the understanding of standard customer 360 model and identifying the subject areas in customer 360 model. Next we maintain a data dictionary, use cases inventory, mapping sheets and perform harmonization of data ingested to data cloud by mapping it to customer 360 data model. For eg: The Party Subject Area in Customer 360 model has Indivdual DMO, Party Identification DMO and Account DMO. Individual DMO relates to Party through IndividualId and PartyIdenficationId; and Individual DMO relates to Account through IndividualId and AccountId.
    Identity Resolution processes several match and reconciliation rules to makes sure that the record is uniquely identified in a unified linked object. The data from unified profiles can be then used for segmentation and activations.
