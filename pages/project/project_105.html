Hosting Options (VirtualMuleys#13 - Deployment Strategies - On-prem to CloudHub Migration path

In the context of Anypoint Platform, there is a runtime plane and control plane. Products and features that let you create, manage and discover mule applications without local setup are included in control plane like Design Center, Exchange, Management Center. Runtime plane is mule runtime along with services like connectors, extensions. Anypoint Studio includes the non-licensed / non-clustered setup of mule runtime in the studio for developers.

In the On-perm model, the control plane is hosted with Mulesoft for the customer org, mule agent connects mule applications running on VMs in customer datacenter to the control plane. In this setup, there are several mule applications run in a single JVM / mule server.

Cloud model is when both control and runtime plane hosted with Mulesoft with an iPaas cloudhub offering which is built on AWS Infrastructure as a Service offering. Mulesoft Cloudhub Offering provides different options like Cloudhub 1.0 (Cloud Workers), Cloudhub 2.0 (Cloud Containers), Runtime Fabric (On-perm containers).

In the migration journeys, there is a hybrid deployment model when the mule applications are deployed both in customer data center and in cloudhub.
