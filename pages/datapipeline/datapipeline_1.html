  • Task-2: CONNECT RDS and EC2 instance on AWS free-tier account.

    1. Create the Key Pair

      aws ec2 create-key-pair --key-name MyKeyPair --query 'KeyMaterial' --output text > MyKeyPair.pem

    2. Add the security group

      aws ec2 create-security-group --group-name my-sg --description "My security group" --vpc-id vpc-xxxxxxxxxxxxxxxxx

    3. Add the ingress rule for SSH

      aws ec2 authorize-security-group-ingress --group-id sg-xxxxxxxxxxxxxxxx --protocol tcp --port 22 --cidr xx.xx.xx.xx/32

    4. Create the EC2 Instance and conncet to instance with ssh

      aws ec2 run-instances --image-id ami-078f95be0757084a3 --count 1 --instance-type t3.micro --key-name MyKeyPair --security-group-ids sg-xxxxxxxxxxxxxxxx --subnet-id subnet-xxxxxxxxxxxxxxxx
      aws ec2-instance-connect ssh --instance-id i-xxxxxxxxxxxxxxx

    5. Create the RDS database instances

      aws rds create-db-instance --db-instance-identifier MySQLForLambda --db-instance-class db.t4g.micro --engine mysql --allocated-storage 20 --master-username admin --master-user-password mypassword --db-name ExampleDB --availability-zone us-east-2b

    6. Setup EC2 connection to RDS Database.

      This will add 2 more security groups, one for RDS as rds-ec2-1 and other for EC@ as ec2-rds-1

    7. Delete the RDS database instance.

      aws rds delete-db-instance --db-instance-identifier MySQLForLambda --skip-final-snapshot

    8. Termimate the EC2 instance.

      aws ec2 terminate-instances --instance-ids i-09a93622030d6fc5e

    9. Delete all 3 security groups.

      aws ec2 delete-security-group --group-id sg-008fe68dc13dd2c06

    10. Delete the Key Pair

      aws ec2 delete-key-pair --key-name MyKeyPair

  • Task-3: CREATE VPC, Public Subnet, Internet Gateway and test an EC2 instance with the security group for web servers. Clean Up Resources.

    3. Create Resources: Create a key pair for SSH access; Find the latest Amazon Linux 2 AMI; Launch a web server in the public subnet; output the public IP address of your web server; Connect to your instances using SSH

      aws ec2 create-key-pair --key-name vpc-tutorial-key --query 'KeyMaterial' --output text > vpc-tutorial-key.pem
      chmod 400 vpc-tutorial-key.pem
      aws ec2 describe-images --owners amazon --filters "Name=name,Values=amzn2-ami-hvm-*-x86_64-gp2" "Name=state,Values=available" --query "sort_by(Images, &CreationDate)[-1].ImageId" --output text
      Note the AMI ID from the output. For this tutorial, we'll use "ami-0ae3c5abab3d5cb90" as an example.
      aws ec2 run-instances --image-id ami-0ae3c5abab3d5cb90 --count 1 --instance-type t3.micro --key-name vpc-tutorial-key --security-group-ids sg-0bc6b079324f82bbf --subnet-id subnet-079925de7c1ac5c62 --associate-public-ip-address --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=WebServer}]
      -- user-data '#!/bin/bash yum update -y yum install -y httpd systemctl start httpd systemctl enable httpd

      ssh -i vpc-tutorial-key.pem ec2-user@3.147.57.81
