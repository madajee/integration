    1. Create the Key Pair

      aws ec2 create-key-pair --key-name MyKeyPair --query 'KeyMaterial' --output text > MyKeyPair.pem

    2. Add the security group

      aws ec2 create-security-group --group-name my-sg --description "My security group" --vpc-id vpc-xxxxxxxxxxxxxxxxx

    3. Add the ingress rule for SSH

      aws ec2 authorize-security-group-ingress --group-id sg-xxxxxxxxxxxxxxxx --protocol tcp --port 22 --cidr xx.xx.xx.xx/32

    4. Create the EC2 Instance and conncet to instance with ssh

      aws ec2 run-instances --image-id ami-078f95be0757084a3 --count 1 --instance-type t3.micro --key-name MyKeyPair --security-group-ids sg-xxxxxxxxxxxxxxxx --subnet-id subnet-xxxxxxxxxxxxxxxx
      aws ec2-instance-connect ssh --instance-id i-xxxxxxxxxxxxxxx

    5. Create the RDS database instances

      aws rds create-db-instance --db-instance-identifier MySQLForLambda --db-instance-class db.t4g.micro --engine mysql --allocated-storage 20 --master-username admin --master-user-password mypassword --db-name ExampleDB --availability-zone us-east-2b

    6. Setup EC2 connection to RDS Database.

      This will add 2 more security groups, one for RDS as rds-ec2-1 and other for EC@ as ec2-rds-1

    7. Delete the RDS database instance.

      aws rds delete-db-instance --db-instance-identifier MySQLForLambda --skip-final-snapshot

    8. Termimate the EC2 instance.

      aws ec2 terminate-instances --instance-ids i-09a93622030d6fc5e

    9. Delete all 3 security groups.

      aws ec2 delete-security-group --group-id sg-008fe68dc13dd2c06

    10. Delete the Key Pair

      aws ec2 delete-key-pair --key-name MyKeyPair

  • Task-5: AWS Lambda with SQS

    1. Setup

      mkdir sqs-tutorial
      cd sqs-tutorial
      touch template-sqs.yaml
      touch lambda-sqs-role.yaml
      touch index.js
      touch send-message.json

    2. template-sqs.yaml

      AWSTemplateFormatVersion: '2010-09-09'
Description: 'A simple CloudFormation template to create a standard Amazon SQS queue.'

Resources:
  MySimpleQueue:
    Type: 'AWS::SQS::Queue'
    Properties:
      QueueName: 'my-simple-standard-queue'

Outputs:
  QueueUrl:
    Description: 'The URL of the created SQS queue.'
    Value: !Ref MySimpleQueue
  QueueArn:
    Description: 'The ARN of the created SQS queue.'
    Value: !GetAtt MySimpleQueue.Arn

    3. lambda-sqs-role.yaml

      AWSTemplateFormatVersion: '2010-09-09'
Description: IAM role for Lambda function to read from SQS queue.

Resources:
  LambdaSQSRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: lambda-sqs-role
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service:
                - lambda.amazonaws.com
            Action:
              - sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AWSLambdaSQSQueueExecutionRole

    4. index.js

      // Copyright Amazon.com, Inc. or its affiliates. All Rights Reserved.
// SPDX-License-Identifier: Apache-2.0
exports.handler = async (event, context) => {
  console.info(JSON.stringify(event));
  console.info("done");
};

    5. send-message.json

      {
  "City": {
    "DataType": "String",
    "StringValue": "Hello City"
  },
  "Greeting": {
    "DataType": "String",
    "StringValue": "Hello, World!"
  },
  "Population": {
    "DataType": "Number",
    "StringValue": "1250800"
  }
}

    6. Create a sqs queue with cloudformation template.

      aws cloudformation create-stack --stack-name simple-sqs-stack --template-body file://template-sqs.yaml

    7. Crete a role for lambda with AWSLambdaSQSQueueExecutionRole permission policy.

      aws cloudformation create-stack --stack-name lambda-sqs-role --template-body file://lambda-sqs-role.yaml --capabilities CAPABILITY_NAMED_IAM

    8. Crete an index.js for the lambda javascript code. Create a deployment package using the following zip command.

      zip function.zip index.js

    9. Create a Lambda function with the role that was created in previous step

      aws lambda create-function --function-name ProcessSQSRecord \
--zip-file fileb://function.zip --handler index.handler --runtime nodejs24.x \
--role arn:aws:iam::xxxxxxxxxxxxxx:role/lambda-sqs-role

    10. Optionally test the lambda with the invoke lambda command

      aws lambda invoke --function-name ProcessSQSRecord --payload file://input.json out --log-type Tail \
--query 'LogResult' --output text --cli-binary-format raw-in-base64-out | base64 --decode

    11. Create an event source mapping between Lambda and SQS queue and list the event source mapping.

      ws lambda create-event-source-mapping --function-name ProcessSQSRecord  --batch-size 10 \
--event-source-arn arn:aws:sqs:us-east-2:xxxxxxxxxxxxx:my-simple-standard-queue
      aws lambda list-event-source-mappings --function-name ProcessSQSRecord

    12. Send a message to the SQS and event source mapping should trigger a lambda.

      aws sqs send-message --queue-url https://sqs.us-east-2.amazonaws.com/xxxxxxxxxxxxx/my-simple-standard-queue --message-body "Information about the largest city in Any Region." --delay-seconds 10 --message-attributes file://send-message.json

    13. Check the Cloudwatch logs.

      aws lambda invoke --function-name ProcessSQSRecord out --log-type Tail \
--query 'LogResult' --output text --cli-binary-format raw-in-base64-out | base64 --decode

    14. Clean Up by deleting the log groups and event logs; Delete the lambda function; deleting the stack for lambda role; deleting the stack for SQS queue.

      aws lambda delete-function --function-name ProcessSQSRecord
      aws cloudformation delete-stack --stack-name lambda-sqs-role
      aws cloudformation delete-stack --stack-name simple-sqs-stack
